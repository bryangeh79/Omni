## Overview

AI-assisted generation is used in two areas:
1. **Onboarding preview** ‚Äî generate AI persona, system prompt, FAQ samples
2. **Knowledge base** ‚Äî future: AI-powered chunking and embedding (Phase 12B)

## Onboarding Preview Generation

### Modes

| Mode | Trigger | Provider call | Safe for tests |
|------|---------|--------------|----------------|
| `DETERMINISTIC_TEMPLATE` | Default | None | Always |
| `AI_GENERATED` | `?mode=ai` + env flag + API key | Real AI provider | No |
| `AI_FALLBACK` | `?mode=ai` but AI fails | None (falls back to template) | Yes |

### Triple Safety Gate (AI mode)

All three conditions must be true to call a real AI provider:

1. Request param `?mode=ai` (explicit opt-in)
2. `OMNI_ENABLE_ONBOARDING_AI=true` in environment
3. Tenant has a configured `AiConfig` record with a valid API key

If any condition fails ‚Üí silently returns `DETERMINISTIC_TEMPLATE` result (or `AI_FALLBACK` if mode=ai was requested but conditions not met).

### Enriched Preview Shape (Phase 12A)

```typescript
  aiPersona:           { name, tone, focus, company }

}
```

### FAQ Samples

Deterministic mode generates 3-5 industry-specific Q&A samples from `INDUSTRY_FAQS` templates embedded in `onboarding.ts`. These are shown in the `/onboarding` Step 4 preview as sample cards.

### Missing Info Warnings

Generated by `buildWarnings()` in `onboarding.ts`. Checks for:
- Missing company name
- Missing industry
- No AI goals selected
- Missing business hours
- No materials text provided

Warnings are shown in the preview step to prompt users to fill in missing data.

## AiProviderFactory

Located at `packages/ai-core/src/provider-factory.ts`.

Providers supported:
- `DRY_RUN` ‚Äî safe default, returns `[AI_DRY_RUN]` prefix (used in all tests)
- `openai` ‚Äî requires `OPENAI_API_KEY` in AiConfig
- `gemini` ‚Äî requires `GEMINI_API_KEY` in AiConfig
- `deepseek` ‚Äî requires `DEEPSEEK_API_KEY` in AiConfig

No real provider is ever called unless `OMNI_ENABLE_ONBOARDING_AI=true` + valid AiConfig + `?mode=ai`.

---

## Limitations (Phase 12A)

- Deterministic templates only ‚Äî no AI personalisation without explicit opt-in
- AI mode requires external provider credentials (not included in demo env)
- No vector embeddings or semantic search yet (Phase 12B)
- System prompt is template-generated, not learned from materials content
